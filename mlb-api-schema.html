MLB Stats API — Data Schema Reference

Base URL: https://statsapi.mlb.com/api/v1  ·  No API key required  ·  Season param: ?season=YYYY

Data Flow & Endpoints

GET /teams

?sportId=1&season=YYYY

  • id integer — MLB team ID
  • teamName string
  • locationNamestring
  • abbreviationstring
  • league.id 103=AL / 104=NL
  • league.namestring
  • division.namestring
  • active boolean

Logo (CDN, no API call)

  • mlbstatic.com/team-logos/{id}.svg
→
GET /standings

?leagueId=103,104&season=YYYY

  • team.id join key
  • wins integer
  • losses integer
  • runsScored integer
  • runsAllowed integer
  • runDifferentialinteger
  • winningPercentagestring
  • divisionRankstring
  • wildCardRankstring
→
DB teams

Supabase · upsert on team_abv

  • mlb_id integer unique
  • name text
  • city text
  • team_abv text unique
  • league text
  • league_abv text
  • division text
  • logo text (URL)
  • wins integer
  • losses integer
  • runs_scored integer
  • runs_allowed integer
  • run_diff integer
GET /teams/{id}/roster

?rosterType=active&hydrate=person(...)

  • person.id integer — player ID
  • person.fullName string
  • person.birthDatestring
  • person.height string
  • person.weight integer
  • person.batSide.codeL/R/S
  • person.pitchHand.codeL/R
  • jerseyNumber string
  • position.abbreviationstring

Headshot (CDN, no API call)

  • img.mlbstatic.com/.../people/{id}/headshot/...
→
GET /stats

?stats=season&group=hitting|pitching
?playerPool=All&sportId=1&limit=2000

  • splits[].player.idjoin key
  • splits[].stat object (see below)
  • splits[].team.id integer

Also used for game logs (Phase 2)

  • ?stats=gameLog&group=hitting
  • splits[].date string
  • splits[].game.gamePkinteger
→
